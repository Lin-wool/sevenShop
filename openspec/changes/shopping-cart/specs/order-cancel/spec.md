## 新增需求

### 需求:取消未处理订单
系统必须允许用户取消处于"待处理"状态的订单。

#### 场景:取消待处理订单
- **当** 用户点击待处理订单的"取消"按钮
- **那么** 系统必须弹出取消原因填写弹窗

#### 场景:填写取消原因
- **当** 弹窗显示后，用户填写取消原因并确认
- **那么** 系统必须将订单状态变更为"已取消"，并保存取消原因

#### 场景:不填写取消原因
- **当** 弹窗显示后，用户不填写原因直接确认
- **那么** 系统必须将订单状态变更为"已取消"，取消原因为空

### 需求:禁止取消已处理订单
系统必须禁止用户取消已处理的订单。

#### 场景:尝试取消已处理订单
- **当** 用户点击已处理订单的"取消"按钮
- **那么** 系统必须提示"该订单已处理，无法取消"

### 需求:禁止取消已取消订单
系统必须禁止用户重复取消已取消的订单。

#### 场景:尝试取消已取消订单
- **当** 用户点击已取消订单的"取消"按钮
- **那么** 系统必须提示"该订单已取消"

### 需求:取消订单后显示
系统必须正确显示已取消的订单状态。

#### 场景:订单列表显示
- **当** 用户查看订单列表
- **那么** 系统必须显示订单状态为"已取消"，并显示取消原因（如果有）

#### 场景:订单详情显示
- **当** 用户查看已取消订单的详情
- **那么** 系统必须显示订单状态为"已取消"和取消原因
