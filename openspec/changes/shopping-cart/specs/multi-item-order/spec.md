## 新增需求

### 需求:批量下单
系统必须支持用户从购物车中选择商品进行批量下单。

#### 场景:从购物车下单
- **当** 用户在购物车页面勾选商品并点击"下单"按钮
- **那么** 系统必须创建包含所有选中商品的订单，并跳转到订单列表

#### 场景:单商品直接下单
- **当** 用户在商品详情页点击"立即下单"按钮
- **那么** 系统必须创建只包含当前商品的订单

### 需求:订单商品明细
系统必须正确存储和显示订单中的所有商品信息。

#### 场景:创建多商品订单
- **当** 用户提交包含3个不同商品的订单
- **那么** 系统必须创建1个订单和3个订单项，每个订单项包含商品ID、规格、数量、单价

#### 场景:显示订单商品列表
- **当** 用户查看订单详情
- **那么** 系统必须显示该订单包含的所有商品及其规格、数量、单价

### 需求:订单总金额计算
系统必须正确计算订单的总金额。

#### 场景:计算订单总价
- **当** 用户提交包含多个商品的订单
- **那么** 系统必须将每个商品的单价乘以数量后求和，作为订单总金额

### 需求:订单收货地址
系统必须要求用户为订单选择收货地址。

#### 场景:选择收货地址
- **当** 用户在下单时选择收货地址
- **那么** 系统必须将选中的地址信息存储到订单中

#### 场景:未选择地址
- **当** 用户提交订单但未选择收货地址
- **那么** 系统必须提示用户选择收货地址
