## 上下文

本项目是一个前后端分离的电商系统（sevenShop），基于 Spring Boot + Vue3。代码审查发现以下问题需要修复：

**当前问题：**
1. 安全性：订单价格可被客户端控制、输入校验缺失、CORS 过度开放
2. 性能：多处 N+1 查询问题
3. 可维护性：Magic Numbers、异常处理不一致、缺少日志

## 目标 / 非目标

**目标：**
- 修复价格篡改安全漏洞
- 解决 N+1 查询性能问题
- 添加完整的输入校验
- 改善代码可维护性（枚举、日志）

**非目标：**
- 不修改邮件密码配置
- 不删除开发接口（DevController）
- 不修改前端购物车持久化方案

## 决策

### 1. 价格安全修复
**决策**：移除 OrderRequest 中的 price 字段，价格始终从商品表获取

**理由**：防止恶意用户篡改价格

**替代方案考虑**：
- 方案A：保留 price 字段，但与服务端比对（不选，仍有篡改风险）
- 方案B：完全移除客户端 price（选）

### 2. N+1 查询优化
**决策**：使用 MyBatis 嵌套查询或 JOIN 查询替代循环内查询

**理由**：减少数据库往返次数

**替代方案考虑**：
- 方案A：使用 @One/@Many 注解（选，代码改动小）
- 方案B：手写 JOIN 查询（可考虑，但不紧急）

### 3. 订单状态管理
**决策**：创建 OrderStatus 枚举类替代 Magic Numbers

**理由**：提升可读性和类型安全

### 4. 异常处理统一
**决策**：创建 BusinessException 替代 RuntimeException

**理由**：统一错误码，便于客户端处理

### 5. CORS 配置
**决策**：使用配置文件管理允许的来源，不硬编码 "*"

**理由**：生产环境需要限制来源

## 风险 / 权衡

- [风险] 价格修复可能影响现有订单创建流程 → 需测试确认
- [风险] N+1 查询优化可能引入 SQL 复杂度 → 使用 MyBatis 注解简化
- [权衡] 枚举改造需改动多处 status 赋值 → 逐步替换，兼容旧数据
