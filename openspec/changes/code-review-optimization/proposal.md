## 为什么

根据代码审查报告，项目存在多个安全性、性能和可维护性问题。这些问题可能导致：
- 安全漏洞（价格篡改、权限控制不足）
- 性能瓶颈（N+1 查询）
- 维护困难（代码重复、缺少日志）

现在进行优化可以提升系统安全性和性能，为后续功能扩展奠定基础。

## 变更内容

### 1. 安全性修复
- **移除客户端价格控制**：订单价格从服务端商品表获取，不接受客户端传入的价格
- **添加输入校验**：为 DTO 添加 `@NotNull`、`@Min`、`@Max` 等校验注解
- **限制 CORS 来源**：生产环境限制 CORS 允许的来源

### 2. 性能优化
- **解决 N+1 查询**：订单列表、商品列表、批量订单创建使用 JOIN 或批量查询
- **添加批量查询优化**：收集商品 ID 后批量查询

### 3. 可维护性改进
- **使用枚举替代 Magic Numbers**：订单状态使用枚举类
- **统一异常处理**：使用 BusinessException 替代 RuntimeException
- **添加日志记录**：关键业务操作添加日志

### 4. 最佳实践改进
- **前端价格从服务端获取**：总价由服务端计算返回
- **添加密码强度校验**：注册时要求密码至少6位

## 功能 (Capabilities)

### 新增功能
无需新增功能，本次是代码优化任务。

### 修改功能
- **订单创建**：价格必须从服务端获取，不接受客户端传入
- **用户列表接口**：需要管理员权限
- **输入校验**：所有 DTO 添加必要的校验注解
