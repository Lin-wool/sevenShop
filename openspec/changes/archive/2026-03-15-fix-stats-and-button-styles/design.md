# 技术设计文档

## 上下文

当前存在以下UI问题：
1. 管理中心数据概览统计数据为0，未正确获取
2. 用户订单页取消按钮位置在金额下方，布局不美观
3. 商品管理页面按钮样式不统一
4. 全局按钮样式不一致

## 目标 / 非目标

**目标：**
- 修复统计数据获取逻辑，正确显示各项数量
- 重新设计用户订单页按钮布局
- 统一商品管理按钮样式
- 建立全局按钮规范

**非目标：**
- 不修改后端API
- 不改变现有功能逻辑

## 决策

### 1. 统计数据修复
- **选择**: 检查API调用方式，确保正确解析响应格式
- **理由**: 数据返回格式可能是分页格式，需要正确提取

### 2. 订单按钮位置
- **选择**: 将取消按钮移至订单卡片底部，与订单金额对齐
- **理由**: 更符合用户操作习惯，视觉效果更好

### 3. 按钮样式统一
- **选择**: 使用el-button组件统一样式，确保对齐
- **理由**: Element Plus组件库保证一致性

## 风险 / 权衡

- **样式冲突** → 使用 scoped 样式避免影响其他组件
- **兼容性问题** → 保持与现有代码风格一致
