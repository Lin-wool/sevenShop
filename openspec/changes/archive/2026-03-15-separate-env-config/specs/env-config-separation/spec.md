## 新增需求

### 需求:环境配置分离
系统必须支持本地开发和生产环境使用不同的配置文件和数据库连接信息。

#### 场景:本地开发环境配置
- **当** 开发者在本地运行应用程序时
- **那么** 系统必须自动使用 `application-local.yml` 中的配置

#### 场景:生产环境配置
- **当** 应用程序部署到生产环境时
- **那么** 系统必须使用 `application-prod.yml` 或环境变量中的配置

### 需求:数据库连接信息
系统必须能够从环境变量读取生产环境的数据库连接信息。

#### 场景:使用环境变量连接数据库
- **当** 生产环境设置了 `SPRING_DATASOURCE_URL` 等环境变量时
- **那么** 系统必须使用这些环境变量覆盖默认配置

### 需求:本地开发默认配置
在没有激活任何 profile 的情况下，系统必须使用本地开发配置。

#### 场景:默认本地配置
- **当** 应用程序启动时没有指定 profile
- **那么** 系统必须使用本地配置文件
