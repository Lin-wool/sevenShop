# 技术设计文档

## 上下文

当前系统存在以下问题：
1. MobileCart.vue 购物车数量输入框失焦后数量未正确保存
2. Mobile.vue 商品购买弹窗只有加减按钮，无直接输入
3. AdminOrders.vue 订单列表缺少下单人信息
4. Admin.vue/MobileAdmin.vue 统计数据获取失败，点击无跳转
5. 缺少用户管理功能

## 目标 / 非目标

**目标：**
- 修复购物车数量输入失焦后数据未保存的问题
- 商品购买弹窗支持直接输入数量
- 订单列表显示下单人昵称
- 统计数据正确显示并支持跳转
- 新增用户管理页面

**非目标：**
- 不修改后端数据库结构
- 不修改订单创建API

## 决策

### 1. 购物车数量输入修复
- **选择**: 使用 v-model 绑定 + @change 事件处理，而非 @input + @blur
- **理由**: v-model 更符合 Vue 响应式原则，确保数据双向绑定

### 2. 商品购买弹窗数量输入
- **选择**: 使用 el-input-number 组件，支持加减和直接输入
- **理由**: 统一组件库，与购物车体验一致

### 3. 订单列表添加用户信息
- **选择**: 后端返回订单时包含 userNickname 字段，前端直接显示
- **理由**: 利用现有订单关联的用户信息，无需额外请求

### 4. 统计功能修复
- **选择**: 修复前端API调用逻辑，正确获取统计数据
- **理由**: 问题在于API响应格式解析，需要适配

### 5. 用户管理
- **选择**: 新建 AdminUsers.vue 页面，添加到路由
- **理由**: 需要展示用户列表和状态切换功能

## 风险 / 权衡

- **API兼容性问题** → 需要确认后端是否有用户列表API
- **统计数据准确性** → 可能需要调整API调用参数
