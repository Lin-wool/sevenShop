# 管理中心统计功能修复规范

## 新增需求

### 需求:统计数据必须正确显示
系统必须在管理中心首页正确显示统计数据。

#### 场景:查看统计数据
- **当** 管理员访问管理中心首页
- **那么** 系统必须显示商品数量、待处理订单数、已完成订单数、用户数量

### 需求:统计卡片必须支持跳转
系统必须支持点击统计卡片跳转到对应的列表页面。

#### 场景:点击商品数量
- **当** 管理员点击商品数量统计卡片
- **那么** 系统必须跳转到商品管理页面

#### 场景:点击待处理订单
- **当** 管理员点击待处理订单统计卡片
- **那么** 系统必须跳转到订单管理页面（筛选待处理）

#### 场景:点击已完成订单
- **当** 管理员点击已完成订单统计卡片
- **那么** 系统必须跳转到订单管理页面（筛选已完成）

#### 场景:点击用户数量
- **当** 管理员点击用户数量统计卡片
- **那么** 系统必须跳转到用户管理页面
