## 新增需求

### 需求:全局异常处理器
系统必须实现全局异常处理机制，统一捕获并记录未处理的异常。

#### 场景:捕获业务异常
- **当** 业务代码抛出 RuntimeException 时
- **那么** 全局异常处理器必须捕获该异常并记录格式化日志

#### 场景:捕获参数验证异常
- **当** 前端传递的参数不符合后端验证规则时
- **那么** 全局异常处理器必须捕获 MethodArgumentNotValidException 并返回友好错误信息

#### 场景:捕获未知异常
- **当** 发生未预期的异常时
- **那么** 全局异常处理器必须记录完整堆栈信息并返回通用错误消息

### 需求:格式化日志输出
系统必须输出格式化的异常日志，包含完整的调试信息。

#### 场景:日志包含请求信息
- **当** 发生异常时
- **那么** 日志必须包含请求方法、请求路径、请求参数

#### 场景:日志包含异常详情
- **当** 发生异常时
- **那么** 日志必须包含异常类型、异常消息、异常堆栈

#### 场景:日志格式统一
- **当** 发生异常时
- **那么** 日志必须按照统一格式输出，便于日志分析和问题排查

### 需求:友好的错误响应
系统必须返回友好的错误信息给前端。

#### 场景:返回错误码和消息
- **当** 发生异常时
- **那么** 响应必须包含错误码和错误消息，前端可以据此展示错误提示
