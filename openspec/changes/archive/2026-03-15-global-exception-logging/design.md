## 上下文

### 背景
- 当前 Spring Boot 后端缺少全局异常处理机制
- Controller 层抛出的异常没有统一的日志记录
- 异常信息仅返回简单的错误消息，缺少调试信息

### 约束
- 保持现有接口返回格式不变（兼容前端）
- 日志输出需符合生产环境可读性要求
- 不能影响正常业务流程性能

## 目标 / 非目标

**目标：**
- 实现全局异常处理器，统一捕获未处理异常
- 格式化日志输出，包含请求路径、请求方法、参数、异常堆栈等信息
- 记录关键业务异常（如订单处理失败）
- 返回友好的错误信息给前端

**非目标：**
- 不修改现有业务逻辑代码
- 不改变接口的正常返回格式

## 决策

**方案：使用 Spring Boot 全局异常处理**

- 使用 `@ControllerAdvice` 注解实现全局异常捕获
- 使用 SLF4J 进行日志记录
- 格式化日志输出，包含：
  - 请求时间
  - 请求方法与路径
  - 请求参数
  - 异常类型与消息
  - 完整堆栈跟踪

**备选方案：**
- 在每个 Controller 中单独 try-catch → 维护成本高，否定
- 使用 Filter 进行异常处理 → 不够精确，否定

## 风险 / 权衡

- 日志量可能增加 → 风险低，可通过日志级别控制
- 堆栈信息可能包含敏感数据 → 仅在开发环境输出完整堆栈，生产环境输出精简信息
