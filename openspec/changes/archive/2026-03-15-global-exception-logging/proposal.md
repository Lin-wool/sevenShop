## 为什么

当前后端服务在发生异常时没有统一的日志输出，导致问题排查困难。当订单列表等接口调用失败时，无法从日志中获取具体的错误信息，严重影响问题定位和修复效率。

## 变更内容

1. **新增全局异常处理器** - 在 Spring Boot 后端添加全局异常捕获机制
2. **统一日志格式输出** - 对所有异常进行格式化输出，包含请求路径、参数、异常信息等关键信息
3. **增强错误响应** - 返回更友好的错误信息给前端

## 功能 (Capabilities)

### 新增功能
- **global-exception-handler**: 全局异常处理器 - 统一捕获并记录后端异常，输出格式化的日志信息

## 影响

- **后端**: 修改 `src/main/java/com/seven/` 目录下的 Java 代码
- **新增文件**: 全局异常处理器类
