## 为什么

目前系统仅支持单商品下单，用户需要一个个添加商品，购物体验不佳。同时用户也无法取消未处理的订单。引入购物车功能和多商品订单支持，可以提升用户购物体验，符合主流电商平台的操作习惯。

## 变更内容

1. **新增购物车功能**
   - 前端 LocalStorage 存储购物车数据
   - 后端提供购物车相关 API（可选，用于数据同步）
   - 支持添加商品、修改数量、删除商品、清空购物车

2. **新增 OrderItem 关联表**
   - 建立 Order 与 Product 的多对多关系
   - 每个订单项记录商品、规格、单价、数量

3. **修改 Order 实体**
   - 增加 totalPrice 字段（订单总金额）
   - 增加 cancelReason 字段（取消原因）
   - 调整 status 枚举：0=待处理，1=已处理，-1=已取消

4. **新增批量下单 API**
   - 支持从购物车选择商品下单
   - 支持单商品直接下单（保持现有逻辑兼容）

5. **新增订单取消功能**
   - 未处理的订单可取消
   - 弹窗让用户填写取消原因（可选）

## 功能 (Capabilities)

### 新增功能
- **shopping-cart**: 购物车功能 - 用户可以将商品加入购物车，支持数量修改、商品删除、清空购物车
- **multi-item-order**: 多商品订单 - 一个订单可包含多个商品，支持批量下单
- **order-cancel**: 订单取消 - 用户可取消未处理的订单，需填写取消原因

### 修改功能
- （无）

## 影响

- **后端**: 新增 Cart 实体和 Cart API（可选），新增 OrderItem 实体，修改 Order 实体，新增批量下单和取消订单 API
- **前端**: 新增购物车页面，修改商品详情页（加入购物车按钮），修改订单确认流程
- **数据库**: 新增 order_items 表，修改 orders 表结构
