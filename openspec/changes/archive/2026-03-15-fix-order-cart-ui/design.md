# UI优化技术设计

## 上下文

当前sevenShop移动端存在以下问题：
1. 订单列表中"取消订单"按钮样式不明显，用户难以识别
2. 订单"已取消"状态没有特殊的样式标签
3. 购物车仅支持数量加减按钮，不支持直接输入
4. 管理员手机端功能不完整，缺少商品管理等功能

## 目标 / 非目标

**目标：**
- 重新设计订单取消按钮样式，使其明显可点击
- 为已取消订单状态添加明显样式标签
- 购物车支持直接输入数量
- 补全手机端管理员功能

**非目标：**
- 不修改后端API
- 不改变现有数据模型
- 不重构现有组件架构

## 决策

### 1. 订单取消按钮样式
- **选择**: 使用带有背景色的实心按钮，带有图标和明确文字
- **理由**: 实心按钮比链接样式更醒目，用户更容易识别为可点击元素

### 2. 订单状态显示
- **选择**: 使用 Element Plus 的 el-tag 组件，不同状态使用不同类型
- **理由**: 保持与现有代码风格一致，利用组件自带样式

### 3. 购物车数量输入
- **选择**: 在现有 el-input-number 基础上添加可编辑输入框模式
- **理由**: el-input-number 组件本身支持直接输入，只需调整配置

### 4. 手机端管理员功能
- **选择**: 复制网页端管理员功能到手机端（商品管理、订单管理、规格模板管理等）
- **理由**: 保持功能一致性，复用现有业务逻辑

## 风险 / 权衡

- **代码重复风险** → 管理员功能代码会在 PC 端和移动端各有一份，但可接受，因为两者布局不同
- **样式一致性** → 需要确保移动端与网页端视觉风格统一
