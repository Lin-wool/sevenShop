# 订单UI优化规范

## 新增需求

### 需求:订单取消按钮必须有明显样式
系统必须确保订单的"取消订单"按钮具有明显的视觉样式，用户能够轻易识别这是一个可点击的按钮。

#### 场景:用户查看待处理订单
- **当** 订单状态为待处理时
- **那么** 取消按钮必须显示为带背景色的按钮样式，而不是文本链接

### 需求:订单状态必须使用状态标签显示
系统必须为不同订单状态使用统一的标签样式进行显示。

#### 场景:显示待处理状态
- **当** 订单状态为待处理（status=0）时
- **那么** 必须显示橙色/警告类型的标签，文字为"待处理"

#### 场景:显示已处理状态
- **当** 订单状态为已处理（status=1）时
- **那么** 必须显示绿色/成功类型的标签，文字为"已处理"

#### 场景:显示已取消状态
- **当** 订单状态为已取消（status=2）时
- **那么** 必须显示红色/危险类型的标签，文字为"已取消"，与其他状态明显区分
