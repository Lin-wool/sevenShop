# 订单和购物车UI优化

## 为什么

当前移动端订单列表存在多个UI/UX问题：订单的取消按钮样式不明显，用户容易忽略；取消状态没有视觉区分；同时购物车仅支持数量加减操作，不支持直接输入；另外手机端管理员功能不完整，影响商家移动办公效率。

## 变更内容

1. **优化订单取消按钮样式** - 重新设计取消订单按钮的视觉样式，使其更醒目、更易识别
2. **优化订单状态显示** - 为"已取消"状态添加明显的样式标签，区分不同订单状态
3. **购物车数量输入优化** - 在购物车中支持直接输入数量，而不仅仅是加减按钮
4. **手机端管理员功能补全** - 确保管理员在手机端可以使用网页端的所有管理功能

## 功能 (Capabilities)

### 新增功能

- `mobile-admin-features`: 手机端完整管理员功能，包括商品管理、订单管理、用户管理等

### 修改功能

- `order-ui`: 订单列表和详情页的UI优化（按钮样式、状态显示）
- `cart-input`: 购物车数量输入方式改进

## 影响

- **前端**: MobileOrders.vue, MobileCart.vue, MobileAdmin.vue 等移动端组件
- **后端**: 无需变更
- **用户体验**: 提升移动端操作便捷性和界面清晰度
